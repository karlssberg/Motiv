**Step 1: Add the static using**

Add `using static Microsoft.CodeAnalysis.CSharp.SyntaxFactory;` to the file. This provides access to `IdentifierName()`, `ObjectCreationExpression()`, `MemberAccessExpression()`, `InvocationExpression()`, `ArgumentList()`, `Argument()`, `SingletonSeparatedList()`, `NormalizeWhitespace()`, and importantly `CarriageReturnLineFeed` for subsequent phases. Remove the existing `using Microsoft.CodeAnalysis.CSharp;` import only if it becomes unused (it provides `SyntaxKind` which IS still needed, so keep it).

**Step 2: Rewrite CreateInternal to accept syntax nodes instead of strings**

The current `CreateInternal(string specName, string modelObjectCreation)` converts syntax nodes to strings, then parses them back. This is the core anti-pattern being eliminated.

Replace with a method that accepts the actual syntax nodes and constructs the expression tree directly. The new internal method signature should be:

```csharp
private static ExpressionSyntax CreateInvocationExpression(
    SimpleNameSyntax specName,
    ExpressionSyntax modelArgument)
```

Note: Use `SimpleNameSyntax` as the specName type because it is the common base of both `IdentifierNameSyntax` and `GenericNameSyntax`, which covers all three overloads.

**Step 3: Build the expression tree leaf-to-root**

The target expression is: `new SpecName().IsSatisfiedBy(modelArgument).Satisfied`

Construct it as:

```csharp
// 1. new SpecName()
var newSpec = ObjectCreationExpression(specName)
    .WithArgumentList(ArgumentList());

// 2. new SpecName().IsSatisfiedBy
var isSatisfiedByAccess = MemberAccessExpression(
    SyntaxKind.SimpleMemberAccessExpression,
    newSpec,
    IdentifierName("IsSatisfiedBy"));

// 3. new SpecName().IsSatisfiedBy(modelArgument)
var isSatisfiedByInvocation = InvocationExpression(
    isSatisfiedByAccess,
    ArgumentList(
        SingletonSeparatedList(
            Argument(modelArgument))));

// 4. new SpecName().IsSatisfiedBy(modelArgument).Satisfied
var satisfiedAccess = MemberAccessExpression(
    SyntaxKind.SimpleMemberAccessExpression,
    isSatisfiedByInvocation,
    IdentifierName("Satisfied"));

return satisfiedAccess.NormalizeWhitespace();
```

**Step 4: Update the three public Create overloads**

Each overload should pass its parameters directly as syntax nodes instead of calling `.ToString()`:

1. `Create(IdentifierNameSyntax specName, ObjectCreationExpressionSyntax modelObjectCreationSyntax)`:
   - Pass `specName` directly as the `SimpleNameSyntax` parameter
   - Pass `modelObjectCreationSyntax` directly as the `ExpressionSyntax` model argument

2. `Create(GenericNameSyntax specName, IdentifierNameSyntax modelVariableName)`:
   - Pass `specName` directly (GenericNameSyntax extends SimpleNameSyntax)
   - Pass `modelVariableName` directly as the model argument

3. `Create(IdentifierNameSyntax specName, IdentifierNameSyntax modelVariableName)`:
   - Pass `specName` directly
   - Pass `modelVariableName` directly as the model argument

**Step 5: Remove the old CreateInternal method**

Delete the old `CreateInternal(string, string)` method entirely. There should be zero `ParseExpression` calls remaining in the file.
