**Algorithm (per CONTEXT.md locked decisions):**

1. **Extract root identifiers** from expression:
   - Use `DescendantNodesAndSelf().OfType<IdentifierNameSyntax>()`
   - Filter to root identifiers only: exclude right-side of member access (`order.Total` -> keep `order`, exclude `Total`)
   - For `IsPatternExpressionSyntax`: only extract from the `.Expression` side (the variable), NOT from the `.Pattern` side (the type). This is a LOCKED decision.
   - Filter out non-variable identifiers using semantic model: only keep IFieldSymbol, IPropertySymbol, ILocalSymbol, IParameterSymbol (follow existing `GetVariablesInExpression` pattern in ConvertToSpecCodeFix.cs line 372-383)

2. **Find common root** via frequency count:
   - Group identifiers by name, order by count descending
   - Use alphabetical tie-breaking (`ThenBy(g => g.Key)`) for determinism
   - If all identifiers are distinct AND there are multiple (no common root), use fallback

3. **Fallback**: When no common root found, return `("Proposition", "Model")` -- LOCKED decision

4. **PascalCase**: Use existing `Capitalize()` extension from `StringExtensions.cs`

5. **Suffixes**: Always append `Proposition` and `Model` -- LOCKED decision (no context-aware suffix adaptation)

6. **Collision detection**: Use `SemanticModel.LookupSymbols(position, name: name)` to check for existing type names. If collision found, append incrementing integers: `OrderProposition` -> `OrderProposition1` -> `OrderProposition2`. LOCKED decision.

**Key implementation details:**
- `IsRootIdentifier` check: `identifier.Parent is not MemberAccessExpressionSyntax memberAccess || memberAccess.Expression == identifier`
- For `IsPatternExpressionSyntax`, walk only `expression.Expression` side, not `expression.Pattern`
- The `DeriveBaseName` method should return the raw identifier name (before PascalCase)
- `EnsureUniqueName` handles collision detection with incrementing counter
