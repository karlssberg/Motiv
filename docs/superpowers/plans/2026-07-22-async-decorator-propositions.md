# Async Decorator Propositions (Core) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `Spec.Build(asyncSpec)` decorator builders to core Motiv — minimal `Create(name)`, explanation `WhenTrue`/`WhenFalse` strings, and metadata `WhenTrue`/`WhenFalse` payloads over an existing `AsyncSpecBase` — so composed async specs can be named and decorated exactly like sync ones.

**Architecture:** Mirror the sync `DecoratorProposition` layer: new async proposition classes that `await` the underlying evaluation and wrap it in the **existing** decorator result types (results are sync-agnostic — async concerns only the *production* of a result). The fluent surface is generated by the Converj generator from new `[FluentTarget]` factory structs, exactly like the sync factories in `src/Motiv/DecoratorProposition/PropositionBuilders/Spec/`.

**Tech Stack:** C# / .NET (core `Motiv` project, multi-TFM — note netstandard2.0-compatible syntax is NOT required here; core targets modern TFMs, match surrounding code), Converj fluent generator, xUnit + Shouldly + AutoFixture (`InlineAutoData`) in `Motiv.Tests`.

## Context for the implementing engineer

- **Sync template files** (read these before each mirroring task):
  - `src/Motiv/DecoratorProposition/MinimalSpecDecoratorProposition.cs` — the minimal (name-only) decorator: evaluates underlying, wraps in `MinimalSpecDecoratorBooleanResult<TMetadata>`.
  - `src/Motiv/DecoratorProposition/SpecDecoratorWithSingleTrueAssertionProposition.cs` — single-string explanation decorator (`PolicyBase<TModel, string>`), wraps in `SpecDecoratorWithSingleTrueAssertionPolicyResult`.
  - `src/Motiv/DecoratorProposition/SpecDecoratorProposition.cs` — metadata decorator (`PolicyBase<TModel, TReplacementMetadata>`).
  - `src/Motiv/DecoratorProposition/PropositionBuilders/Spec/MinimalSpecDecoratorFactory.cs`, `.../PropositionFactory.cs`, `.../MultiAssertionSpecExplanationWithNamePropositionFactory.cs` — the `[FluentTarget(typeof(Motiv.Spec))]` factory structs the generator turns into `Spec.Build(...)` chains.
  - `src/Motiv/Shared/SpecBuildOverloads.cs` — `[FluentMethodTemplate]` overload set for the `Build` entry point.
- **Async plumbing you will reuse:**
  - `AsyncSpecBase<TModel, TMetadata>` (`src/Motiv/AsyncSpecBase.cs`): derive from it; implement `MatchesAsync(TModel, CancellationToken)` and `protected override ValueTask<BooleanResultBase<TMetadata>> EvaluateSpecAsync(TModel, CancellationToken)`. Await the underlying via `underlying.EvaluateSpecAsyncInternal(model, ct)` (internal — this code lives in core, so it is accessible).
  - `AsyncPolicyBase<TModel, TMetadata>` (`src/Motiv/AsyncPolicyBase.cs`): implement `protected override ValueTask<PolicyResultBase<TMetadata>> EvaluatePolicyAsync(TModel, CancellationToken)`; await policies via `EvaluatePolicyAsyncInternal`.
  - Everything returns `ValueTask` (post-#78). Use `.ConfigureAwait(false)` on awaits, matching `AsyncSpecBase` internals.
- **Semantics contract** (from CLAUDE.md, must hold for async exactly as sync):
  - `Create("name")` on a decorated spec → assertions are `"name == true"` / `"name == false"`; WhenTrue/WhenFalse strings demote to `Values`.
  - `.WhenTrue("t").WhenFalse("f").Create()` (unnamed) → the strings ARE the assertions; statement is the WhenTrue string.
  - Metadata payloads → `Values` carry the payloads; assertions are `"name == true"`/`"name == false"`.
  - Singular WhenTrue/WhenFalse → **policy** (async: `AsyncPolicyBase`).
- **Parity oracle:** every test asserts the async result equals the sync equivalent built from the same underlying logic — same `Satisfied`, `Reason`, `Assertions`, `Values`, `Justification`. See `src/Motiv.Tests/AsyncExplanationPropositionTests.cs` and `MixedSyncAsyncCompositionTests.cs` for the established comparison style.
- If the generator's conventions fight you on any factory, inspect the generated output under `obj/` (search for the sync factory's generated partial) to see the expected wiring before adjusting.

```csharp
namespace Motiv.DecoratorProposition;

internal sealed class AsyncMinimalSpecDecoratorProposition<TModel, TMetadata>(
    AsyncSpecBase<TModel, TMetadata> underlyingSpec,
    ISpecDescription description)
    : AsyncSpecBase<TModel, TMetadata>
{
    private readonly SpecBase[] _underlying = [underlyingSpec];

    public override IEnumerable<SpecBase> Underlying => _underlying;

    public override ISpecDescription Description => description;

    public override ValueTask<bool> MatchesAsync(TModel model, CancellationToken cancellationToken = default) =>
        underlyingSpec.MatchesAsync(model, cancellationToken);

    protected override async ValueTask<BooleanResultBase<TMetadata>> EvaluateSpecAsync(
        TModel model,
        CancellationToken cancellationToken)
    {
        var underlyingResult = await underlyingSpec
            .EvaluateSpecAsyncInternal(model, cancellationToken)
            .ConfigureAwait(false);

        return new MinimalSpecDecoratorBooleanResult<TMetadata>(underlyingResult, description);
    }
}
```

```csharp
internal static class AsyncSpecBuildOverloads

    internal static AsyncSpecBase<TModel, TMetadata> Build<TModel, TMetadata>(AsyncSpecBase<TModel, TMetadata> spec) =>
        spec.ThrowIfNull(nameof(spec));
```

```csharp
using Converj.Attributes;
using Motiv.Shared;

namespace Motiv.DecoratorProposition.PropositionBuilders.AsyncSpec;

/// <summary>
/// A factory for renaming an existing asynchronous proposition, mirroring the synchronous
/// <see cref="Motiv.DecoratorProposition.PropositionBuilders.Spec.MinimalSpecDecoratorFactory{TModel,TMetadata}" />.
/// </summary>
/// <param name="spec">The asynchronous proposition to decorate.</param>
/// <typeparam name="TModel">The type of the model.</typeparam>
/// <typeparam name="TMetadata">The type of the underlying metadata associated with the proposition.</typeparam>
[FluentTarget(typeof(Motiv.Spec), TerminalMethod = TerminalMethod.None)]
public readonly partial struct AsyncMinimalSpecDecoratorFactory<TModel, TMetadata>(
    [MultipleFluentMethods(typeof(AsyncSpecBuildOverloads))]AsyncSpecBase<TModel, TMetadata> spec)
{
    /// <summary>Creates an asynchronous proposition named with the propositional statement provided.</summary>
    /// <param name="statement">The proposition statement of what the proposition represents.</param>
    /// <returns>An asynchronous proposition for the model.</returns>
    /// <exception cref="ArgumentException">Thrown when <paramref name="statement"/> is null, empty or whitespace.</exception>
    public AsyncSpecBase<TModel, TMetadata> Create(string statement) =>
        new AsyncMinimalSpecDecoratorProposition<TModel, TMetadata>(
            spec,
            new SpecDescription(statement.ThrowIfNullOrWhitespace(nameof(statement)),
                spec.Description));
}
```
